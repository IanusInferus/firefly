本格式用于存储图形文本。

本格式的实际内容在fd、bmp、txt文件中。

fd和bmp请参照FD文件格式表(FD.zh.htm)，fd文件的编码应为UTF-16L。

bmp另请参照BMP文件格式表(BMP.zh.htm)，注意应使用RGB颜色表示，一般黑表示字，白表示背景。

txt文件为Agemo格式，编码应为UTF-16L，采用特殊转义语义。

该特殊转义语义如下：

EscapedString ::= EscapedUTF16Char+ -> GetUTF16StringAsUTF32String(EscapedUTF16Char+)

EscapedChar ::= "\0" -> U+0000

        | "\a" -> U+0007

        | "\b" -> U+0008

        | "\t" -> U+0009

        | "\n" -> U+000A

        | "\v" -> U+000B

        | "\f" -> U+000C

        | "\r" -> U+000D

        | "\u" UTF16Code:[0-9A-Fa-f]{4} -> GetUTF16CharAsUTF16String(UTF16Code)

        | "\U" UTF32Code:[0-9A-Fa-f]{5} -> GetUTF32CharAsUTF16String(UTF32Code)

        | "\g{" CustomCode:[0-9A-Fa-f]+ "}" -> GetStringCode(CustomCode)

        | UTF16Char:. -> UTF16Char

注意:

1、所有的整数数据类型都是little-endian的。

2、每个段是16字节对齐的。

3、为了便于网络传输，可能使用 RFC 1952 （GZIP 文件格式规范 4.3） 对整个文件进行压缩。但也可以不使用。

   使用压缩时应对整个文件进行压缩，并在起始位置增加标识符"LOCC"。

   .Net Framework 2.0 的 System.IO.Compression.GZipStream 类实现了此压缩。
